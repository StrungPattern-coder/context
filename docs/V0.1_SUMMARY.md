## ðŸŽ¨ Design Decisions Made

### 1. Version 0.1 (Not 1.0)

**Rationale:** Core is complete but not yet proven at scale. v0.1 signals:
- Working, tested, usable
- Not yet production-hardened
- API may evolve (but with deprecation warnings)

### 2. Apache 2.0 for Core (Not MIT)

**Rationale:**
- Trust requires transparency (must be open)
- Patent grant protects users
- Enterprise-friendly
- Attribution preserved

### 3. Docker as Primary Distribution

**Rationale:**
- Universal (Mac, Linux, Windows, cloud)
- Private (data stays local)
- Deterministic
- No vendor lock-in
- Developer-familiar

### 4. /api/v0 URL Structure

**Rationale:**
- Explicit versioning in URL
- v0 = "may break, but with notice"
- v1 = "stable, backward compatible"
- Easy to run both versions side-by-side

### 5. Thin Clients

**Rationale:**
- All logic in server
- Clients are dumb pipes
- Easy to update (no breaking changes)
- Works everywhere
